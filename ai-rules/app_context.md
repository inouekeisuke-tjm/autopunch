## 1. App Overview (プロダクト概要)
- **App Name**: AutoPunch (オートパンチ)
- **Language**: Japanese
- **Tagline**: 打刻を、記憶から消す。勤怠管理の完全自動化ツール
- **Core Value**: 毎日繰り返す単調な打刻作業をゼロにし、本来集中すべき業務へリソースを割くための「時間の解放」
- **Platform**: Web App (PWA対応)

## 2. Target Persona (ターゲットユーザー)
- **Primary User**: 毎日の勤怠打刻作業を煩雑に感じているビジネスパーソン
- **Mindset**: 効率化を好み、PCの起動状況に左右されずに定時ルーチンを完遂したい。複雑な操作や管理は避けたい。
- **Pain Points (解決したい悩み)**:
  - 毎日の出退勤ボタンを押すという行為自体がストレス。
  - 忙しさのあまり、打刻を忘れてしまうことがある。
  - PC起動が必要な常駐型アプリは、PCを閉じた状態での打刻に対応できない。

## 3. User Journey & Key Features (主な機能と体験)
### Core Loop (ユーザーが毎日やること)
1. **Trigger**: 指定時間（8:40 / 21:00）の到達
2. **Action**: システムがバックグラウンドで勤怠サイトへアクセスし、ログインして打刻を実行
3. **Reward**: 打刻完了を知らせるメールを受信し、タスクから解放されたことを実感する

### Must-Have Features (必須機能)
- **セキュアなログイン情報管理**: 勤怠システム用ID・パスワードの暗号化保存
- **スケジュール管理機能**: 打刻時間の変更・一時停止ができる管理ダッシュボード
- **エラーハンドリング＆通知**: 打刻失敗時に即時メール通知を行う機能

## 4. Key UX Psychology Strategies (適用するUX心理学)
- **デフォルト効果 (Default Effect)**:
  - 8:40と21:00を初期値として設定。ユーザーは最小限の初期設定のみで自動化の恩恵を享受できる状態を作る。
- **自己効力感 (Self-Efficacy)**:
  - 成功時に完了通知メールを送ることで、システムが正しく機能しているという確信を与え、ユーザーの心理的安心感を担保する。

## 5. Design Tone & Voice (トーン＆マナー)
- **Mood**: Minimal, Secure, Efficient
- **Voice**: 信頼感のある敬語。過度な装飾を排し、機能美を重視。
- **Emoji Usage**: 通知メール内などで使用し、堅苦しすぎない親しみやすさを付与。
